# DreamPath 3단계 게임 설계서
# 전우(戰友) 시스템 · 개인/팀 프로젝트 · 문의 공유 생태계

> **"혼자 가면 빠르고, 함께 가면 멀리 간다."**
> 같은 꿈을 가진 전우와 함께, 기획부터 배포까지 완주한다.

---

## 0. 설계 철학 — 3단계가 필요한 이유

```mermaid
flowchart LR
    subgraph Problem["혼자 하면 생기는 문제"]
        P1["막막하다<br> '어디서부터 시작하지?'"]
        P2["포기한다<br> '혼자라서 힘들어'"]
        P3["증명이 없다<br> '내가 뭘 했는지 모르겠어'"]
    end

    subgraph Solution["3단계 전우 시스템 해결"]
        S1["같은 목표 전우<br> → 방향이 보인다"]
        S2["공유 일정 + 책임감<br> → 끝까지 완주"]
        S3["배포까지 트래킹<br> → 완성된 포트폴리오"]
    end

    Problem --> Solution

    style Problem fill:#E74C3C,color:#fff
    style Solution fill:#1ABC9C,color:#fff
```

### 핵심 설계 원칙

| 원칙 | 내용 | 이유 |
|------|------|------|
| **전우(戰友) 관계** | 팀원이 아닌 전우. 같은 전쟁을 싸우는 동료 | 게임적 몰입감 + 책임감 |
| **코딩은 외부에서** | 플랫폼은 기획/일정/문의만 관리. 실제 개발은 외부 도구 | 진입장벽 최소화 |
| **배포가 완성 기준** | MVP 배포까지 트래킹. "만들다 멈춤"은 완성이 아님 | 커리어 증명 가능 결과물 |
| **문의 = 생명줄** | 초보 개발자의 첫 질문을 환영하는 문화 | 이탈 방지 핵심 장치 |
| **개인도, 팀도** | 1인 프로젝트 + 팀 프로젝트 모두 동등하게 지원 | 성격·상황 다양성 반영 |
| **기록이 포트폴리오** | 기획서, 일정표, 문의 해결 과정이 자동으로 포트폴리오 | 별도 작성 없음 |

## 2. 전우 매칭 시스템

### 2.1 매칭 기준 비교표

| 매칭 요소 | 가중치 | 판단 기준 | 예시 |
|----------|--------|----------|------|
| **커리어 방향 일치** | 40% | 관심 직업 카테고리 동일 | UX디자이너 지망생끼리 |
| **RIASEC 유형 유사** | 25% | 동일하거나 보완적 유형 | 예술형 + 탐구형 조합 |
| **프로젝트 유형 선호** | 20% | 개인/팀, 프로젝트 종류 선택 | "앱 만들고 싶어" |
| **학년/나이 근접** | 10% | ±2년 이내 | 중3 ~ 고2 |
| **활동 시간대** | 5% | 주말형/평일형 | 주말 오후 주로 활동 |

### 2.2 전우 매칭 흐름

```mermaid
sequenceDiagram
    participant U as 유저
    participant App as DreamPath
    participant Match as 매칭 엔진
    participant Clue as 클루

    U->>App: 3단계 시작 버튼 클릭
    App->>U: "팀으로? 혼자?"
    
    alt 팀 선택
        U->>App: 팀 프로젝트 선택
        App->>Match: RIASEC + 커리어 방향 + 프로젝트 유형 전달
        Match->>App: 매칭 후보 3~5명 반환
        App->>U: 전우 후보 카드 표시 (프로필 + 매칭도)
        U->>App: 전우 수락/거절 선택
        App->>Clue: 클루 생성 (채팅방 + 프로젝트 보드 자동 생성)
        Clue->>U: "전우들이 기다리고 있어! 첫 인사 해볼래?"
    else 솔로 선택
        U->>App: 1인 프로젝트 선택
        App->>U: 솔로 트랙 시작 (개인 프로젝트 보드 생성)
        App->>U: "혼자지만, 문의 게시판 전우들이 응원해!"
    end
```

## 4. 프로젝트 시스템 (개인 + 팀)

### 4.1 프로젝트 유형 비교표

| 구분 | 1인 솔로 프로젝트 | 팀 클루 프로젝트 |
|------|----------------|----------------|
| **인원** | 1명 | 3~5명 |
| **기간** | 자유 (권장 8주) | 시즌 기반 (10주) |
| **기획** | 개인 기획서 템플릿 | 공동 기획서 + 역할 분담 |
| **일정** | 개인 칸반 보드 | 공유 칸반 보드 |
| **문의** | 전체 Q&A 게시판 | 클루 Q&A + 전체 Q&A |
| **배포** | 개인 배포 + 인증 | 팀 배포 + 클루 인증 |
| **포트폴리오** | 개인 포트폴리오 자동 생성 | 개인 + 팀 포트폴리오 모두 생성 |
| **DreamFestival** | 개인 출품 가능 | 클루 출품 가능 |
| **전우 XP** | 개인 XP만 적립 | 클루 공유 XP + 1.5배 보너스 |
