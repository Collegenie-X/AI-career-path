# Expo 개발 키보드 단축키 가이드

## 📱 기본 개발 서버 명령어

### 터미널에서 `npm start` 실행 후 사용 가능한 키보드 단축키:

| 키 | 기능 | 설명 |
|---|---|---|
| `a` | Android 실행 | Android 에뮬레이터에서 앱 실행 |
| `i` | iOS 실행 | iOS 시뮬레이터에서 앱 실행 |
| `w` | Web 실행 | 웹 브라우저에서 앱 실행 |
| `r` | 새로고침 | 앱 새로고침 (모든 플랫폼) |
| `m` | Metro 메뉴 토글 | 개발자 메뉴 표시/숨김 |
| `d` | 개발자 도구 | 개발자 메뉴 열기 |
| `j` | Debugger 열기 | Chrome DevTools 디버거 실행 |
| `c` | 캐시 클리어 | Metro 번들러 캐시 클리어 |
| `s` | 서버 전환 | LAN/Local 서버 모드 전환 |
| `?` | 도움말 | 사용 가능한 명령어 표시 |

## 🔄 앱 새로고침 방법

### 1. 터미널에서
```bash
# 개발 서버가 실행 중일 때
r  # 새로고침
```

### 2. 디바이스/시뮬레이터에서
- **iOS**: `Cmd + R` (시뮬레이터) 또는 디바이스 흔들기
- **Android**: `RR` (빠르게 두 번 R 키) 또는 `Cmd + M` (에뮬레이터)
- **물리 디바이스**: 디바이스 흔들기

## 🛠 개발자 메뉴 열기

### iOS 시뮬레이터
- `Cmd + D` 또는 `Cmd + Ctrl + Z`
- Hardware > Shake Gesture

### Android 에뮬레이터
- `Cmd + M` (Mac) / `Ctrl + M` (Windows/Linux)
- `F2` 키

### 물리 디바이스
- 디바이스 흔들기
- 3손가락으로 화면 터치 (iOS)

## 📱 시뮬레이터/에뮬레이터 단축키

### iOS 시뮬레이터
| 키 | 기능 |
|---|---|
| `Cmd + Shift + H` | 홈 버튼 |
| `Cmd + Shift + H` (두 번) | 앱 스위처 |
| `Cmd + L` | 디바이스 잠금 |
| `Cmd + Shift + M` | 메모리 경고 시뮬레이션 |
| `Cmd + K` | 키보드 토글 |
| `Cmd + 1,2,3,4` | 디바이스 크기 조절 |

### Android 에뮬레이터
| 키 | 기능 |
|---|---|
| `Cmd + H` | 홈 버튼 |
| `Cmd + B` | 뒤로 가기 |
| `Cmd + O` | 최근 앱 |
| `Cmd + P` | 전원 버튼 |
| `Cmd + V` | 볼륨 조절 |

## 🔧 문제 해결 단축키

### 캐시 클리어
```bash
# 터미널에서 개발 서버 실행 중
c  # Metro 캐시 클리어

# 또는 완전 캐시 클리어
npm start -- --clear-cache
# 또는
npx expo start --clear
```

### 앱 데이터 리셋 (우리 앱 전용)
1. 스플래시 화면에서 우상단 🛠 아이콘 클릭
2. "온보딩 리셋" 또는 "전체 데이터 삭제" 선택

## 🚀 빠른 개발 워크플로우

### 1. 앱 시작
```bash
cd app/dreampath-app
npm start
```

### 2. 플랫폼 선택
- `i` - iOS 시뮬레이터 (추천)
- `a` - Android 에뮬레이터
- `w` - 웹 브라우저

### 3. 개발 중
- 코드 수정 → 자동 새로고침 (Fast Refresh)
- 문제 발생 시 → `r` (새로고침)
- 캐시 문제 시 → `c` (캐시 클리어)

### 4. 디버깅
- `j` - Chrome DevTools 열기
- `Cmd + D` - 개발자 메뉴 (디바이스에서)
- Network 탭에서 API 호출 확인

## 📝 유용한 팁

### 1. 빠른 테스트를 위한 데이터 리셋

**방법 A: 개발자 메뉴 사용 (추천)**
1. 스플래시 화면 → 🛠 아이콘 클릭
2. "📊 저장된 데이터 확인" → 현재 상태 확인
3. "🔄 온보딩 리셋" 또는 "🗑️ 전체 데이터 삭제" 선택
4. **중요**: 앱을 완전히 종료 (백그라운드에서도 제거)
5. 앱을 다시 실행

**방법 B: 터미널에서 캐시 클리어 (가장 확실)**
```bash
# 개발 서버 실행 중
c  # Metro 캐시 클리어
r  # 앱 새로고침

# 또는 완전 재시작
Ctrl + C  # 서버 종료
npm start -- --clear  # 캐시 클리어하고 재시작
```

**방법 C: 시뮬레이터/에뮬레이터 앱 데이터 삭제**
- iOS: 시뮬레이터에서 앱 길게 누르기 → 앱 삭제 → 다시 설치
- Android: 설정 → 앱 → DreamPath → 저장공간 → 데이터 삭제

### 2. 성능 모니터링
- 개발자 메뉴 → "Perf Monitor" → FPS/메모리 사용량 확인

### 3. 네트워크 디버깅
- 개발자 메뉴 → "Debug" → Chrome DevTools → Network 탭

### 4. 레이아웃 디버깅
- 개발자 메뉴 → "Show Inspector" → 요소 검사 모드

## ⚠️ 주의사항

1. **물리 디바이스 사용 시**: 같은 Wi-Fi 네트워크에 연결되어 있어야 함
2. **캐시 문제**: 이상한 동작 시 `c` (캐시 클리어) 먼저 시도
3. **Metro 서버**: 터미널을 닫으면 개발 서버도 종료됨
4. **Hot Reload**: 일부 변경사항은 앱 재시작이 필요할 수 있음
