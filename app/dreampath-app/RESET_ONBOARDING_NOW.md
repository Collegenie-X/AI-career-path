# 🚨 지금 바로 온보딩으로 돌아가는 방법

## 현재 상황
`npm start -- --clear`를 실행해도 여전히 홈 화면으로 넘어갑니다.

## ✅ 해결 방법 (100% 확실)

### 방법 1: iOS 시뮬레이터에서 앱 삭제 (가장 확실) ⭐⭐⭐

1. **iOS 시뮬레이터를 실행 중인 상태에서**
2. DreamPath 앱 아이콘을 **길게 누르기** (2-3초)
3. **"앱 삭제"** 또는 **"-"** 버튼 클릭
4. 삭제 확인
5. 터미널에서:
   ```bash
   npm start
   i  # iOS 시뮬레이터에 앱 재설치 및 실행
   ```

### 방법 2: 터미널 명령어로 삭제

**iOS 시뮬레이터가 실행 중인 상태에서:**
```bash
# 1. 앱 삭제
npm run reset:ios

# 2. 개발 서버 시작
npm start

# 3. iOS 실행
i
```

**Android 에뮬레이터가 실행 중인 상태에서:**
```bash
# 1. 앱 데이터 삭제
npm run reset:android

# 2. 개발 서버 시작
npm start

# 3. Android 실행
a
```

### 방법 3: 시뮬레이터 완전 재시작

**iOS:**
```bash
# 1. 시뮬레이터 종료
# Device → Erase All Content and Settings...

# 2. 시뮬레이터 재시작
# 3. 개발 서버 시작
npm start
i
```


---

## 🔍 왜 이 문제가 발생하나요?

AsyncStorage는 앱 데이터로 저장되므로:
- Metro 캐시를 지워도 (`--clear`) AsyncStorage는 남아있음
- 앱을 재설치하지 않으면 이전 데이터가 계속 유지됨

**해결책**: 앱 자체를 삭제하고 재설치해야 합니다.

## 📱 시뮬레이터 상태 확인

### iOS 시뮬레이터가 실행 중인지 확인:
```bash
xcrun simctl list devices | grep Booted
```

출력이 있으면 시뮬레이터가 실행 중입니다.

### 현재 설치된 앱 확인:
```bash
xcrun simctl listapps booted | grep dreampath
```

## 🛠 개발자 메뉴는 언제 사용하나요?

개발자 메뉴 (🛠)는 **앱을 삭제하지 않고** 데이터만 리셋하고 싶을 때 사용합니다.

하지만 **앱을 재시작해야 적용**되므로, 시뮬레이터에서 앱을 직접 삭제하는 것이 더 빠릅니다.
