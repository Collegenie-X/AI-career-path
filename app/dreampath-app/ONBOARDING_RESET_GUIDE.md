# 온보딩 화면으로 돌아가는 방법

## 🚨 문제: 홈 화면으로 바로 넘어가는 경우

앱이 이미 온보딩을 완료한 것으로 인식하여 홈 화면으로 바로 이동합니다.

---

## ✅ 해결 방법 (3가지)

### 방법 1: 개발자 메뉴 사용 (가장 쉬움) ⭐

1. **앱 실행** → 스플래시 화면에서 **우상단 🛠 아이콘 클릭**
2. **"📊 저장된 데이터 확인"** 클릭 → 현재 상태 확인
3. **"🔄 온보딩 리셋"** 클릭
4. 확인 버튼 클릭
5. **⚠️ 중요**: 앱을 완전히 종료
   - **iOS**: 앱 스와이프 업으로 종료
   - **Android**: 최근 앱 목록에서 제거
6. 앱을 다시 실행 → 온보딩 화면부터 시작!

---

### 방법 2: 터미널에서 캐시 클리어 (가장 확실) ⭐⭐

```bash
# 1. 개발 서버 종료
Ctrl + C

# 2. 캐시 클리어하고 재시작
npm start -- --clear

# 3. iOS 시뮬레이터 실행
i
```

이 방법은 Metro 번들러 캐시와 AsyncStorage를 모두 초기화합니다.

---

### 방법 3: 시뮬레이터에서 앱 삭제 (완전 초기화)

#### iOS 시뮬레이터
1. 시뮬레이터에서 DreamPath 앱 아이콘 **길게 누르기**
2. **"앱 삭제"** 선택
3. 터미널에서 `i` 입력 → 앱 재설치 및 실행

#### Android 에뮬레이터
1. 설정 → 앱 → DreamPath
2. **저장공간** → **데이터 삭제**
3. 터미널에서 `a` 입력 → 앱 재실행

---

## 🔍 현재 상태 확인하기

앱이 온보딩을 완료했는지 확인하려면:

1. 스플래시 화면 → 🛠 클릭
2. **"📊 저장된 데이터 확인"** 클릭
3. 팝업에서 확인:
   - `온보딩: 완료` → 홈 화면으로 이동
   - `온보딩: 미완료` 또는 `사용자: 없음` → 온보딩 화면으로 이동

---

## 💡 개발 팁

### 온보딩 플로우를 자주 테스트해야 한다면:

```bash
# 터미널에서 개발 서버 실행 중
c  # 캐시 클리어
r  # 새로고침
```

이 조합을 사용하면 빠르게 초기화할 수 있습니다.

### 개발자 메뉴 단축키

- **스플래시 화면**: 우상단 🛠 클릭
- **메뉴 옵션**:
  - 📊 저장된 데이터 확인
  - 🔄 온보딩 리셋
  - 🗑️ 전체 데이터 삭제

---

## ⚠️ 주의사항

1. **앱 재시작 필수**: 개발자 메뉴에서 리셋 후 반드시 앱을 완전히 종료하고 다시 실행해야 합니다.
2. **백그라운드 제거**: 단순히 홈 버튼을 누르는 것이 아니라, 앱 스위처에서 완전히 제거해야 합니다.
3. **개발 서버 재시작**: 가장 확실한 방법은 개발 서버를 재시작하는 것입니다.

---

## 🎯 추천 워크플로우

온보딩 → 퀴즈 플로우를 테스트할 때:

```bash
# 1. 서버 재시작 (캐시 클리어)
Ctrl + C
npm start -- --clear
i

# 2. 온보딩 완료 후 다시 테스트하려면
# 앱에서: 🛠 → 온보딩 리셋 → 앱 종료 → 재실행

# 3. 또는 터미널에서
c  # 캐시 클리어
r  # 새로고침
```

이제 온보딩 화면부터 자유롭게 테스트할 수 있습니다! 🚀
