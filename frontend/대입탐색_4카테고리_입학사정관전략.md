> **목표:** 진로 키워드와 학습 루틴의 기반 잡기

```

├── 진로 키워드 2개 확정
│   ├── 직업/학과 탐색 자료 주 1회 정리

│   └── 학기 말: 키워드 유지/변경 판단
├── 교과 기반 탐구 루틴
│   ├── 과목별 미니 탐구 주제 (한 달 1개)

│   └── 교사 피드백 반영 후 재정리

**실전 예시:**

**고1 사정관 체크리스트:**

| 항목 | 체크 포인트 |

### 🤖 AI 프로젝트 학습

**핵심 원칙:** AI는 이제 기본 옵션. **'AI 쓴다'는 차별점이 아니다.** 중요한 건 기획·실행·디버깅·관리 과정 역량이다.

#### 입학사정관이 AI 프로젝트에서 보는 것

| 평가 항목 | 사정관 관점 | 실전 증빙 방법 |
|---|---|---|
| **문제 정의** | AI 도구 이름보다 문제 정의의 질 | 해결 문제를 한 문장으로 압축 + 성공 기준 2개 |
| **탐구 깊이** | 프롬프트 실험·오류 수정 로그 | 실험 로그, 오류 수정 근거, 선택 기준 기록 |
| **협업 역량** | 역할 분담·의사결정 근거 | 역할 분담표 + 커밋 로그 + 회고 기록 |
| **문제해결력** | 개선 전후 지표 제시 | 정확도·처리시간·오류율 수치 비교 |
| **책임감** | AI 윤리·출처·저작권 점검 | 데이터 출처·저작권 기준 명시 |

#### AI 프로젝트 4단계 프로세스

```
Step 1. 기획 (문제 정의)
├── 해결하려는 문제 한 문장 압축
├── 성공 기준 2개 설정 (수치 포함)
└── AI 도구 선택 근거 문서화

Step 2. 실행 (구현 + 프롬프트 실험)
├── 프롬프트 실험 로그 기록
├── 모델 비교 실험 결과 정리
└── 팀 역할 분담 + 의사결정 기록

Step 3. 디버깅 (오류 → 개선)
├── 오류 재현 조건 문서화
├── 원인 가설 → 수정 → 재테스트
└── 개선 전후 지표 비교 (초기값·목표값·최종값)

Step 4. 관리 (마무리 + 회고)
├── 주간 계획 대비 실행 차이 분석
├── AI 윤리·데이터 출처 점검
└── 면접 답변 구조 준비 (60초·3분 버전)
```

**실전 예시 (학년별):**

| 학년 | 프로젝트 규모 | 구체적 예시 |
|---|---|---|
| 고1 | 2주 단위 소규모 | 챗GPT로 독서 요약 → 오류 찾기 → 개선 반복 |
| 고2 | 팀 프로젝트 (리딩) | AI 날씨 예보 정확도 개선 프로젝트 + 전후 수치 비교 |
| 고3 | 대표 사례 정리 | 기존 AI 프로젝트 1~2개 압축 → 서류·면접 메시지 일치 |

### 🎓 입학사정관이 전공 적합성을 판단하는 기준

|---|---|---|
| **연결성** | 교과·탐구·활동이 같은 진로를 향하는가 | 진로 키워드 중심으로 활동 재배치 |
| **심화 수준** | 학년이 올라갈수록 깊이가 증가하는가 | 고1 기초→고2 심화→고3 통합 흐름 설계 |
| **자기화** | 학생의 언어로 설명 가능한 경험인가 | 외부 도움 최소화, 본인 판단 과정 기록 |
| **적합성** | 전형이 기대하는 인재상과 일치하는가 | 대학별 전형요강 원문 기준 분기 갱신 |

### 📝 세특 작성 전략

#### 사정관이 세특에서 체크하는 항목

```
세특 평가 흐름
├── 수업 행동 (기본 태도)
│   ├── 질문의 수준 (교과 개념 넘어 확장됐는가?)
│   ├── 수업 참여 방식 (능동적 탐구인가?)
│   └── 피드백 반영 흔적 (개선 의지가 보이는가?)
├── 확장 탐구 (심화 역량)
│   ├── 교과 개념 → 실제 문제에 적용 (학업 역량 연결)
│   ├── 데이터·현상 기반 해석 (근거 기반 사고)
│   └── 한계 인식 + 후속 가설 제시 (연구 완성도)
└── 성장 증거 (변화 추적)
    ├── 고1 기초 탐구 태도
    ├── 고2 심화 분석
    └── 고3 통합 성과 + 성찰
```

#### 세특 학년별 설계 전략

| 학년 | 세특 핵심 역할 | 구체적 기록 내용 |
|---|---|---|
| 고1 | 기초 탐구 태도 | 질문 내용, 탐구 시작 동기, 관심 분야 탐색 |
| 고2 | 심화 분석 | 데이터 해석, 실험 결과, 팀 내 역할과 기여 |
| 고3 | 통합 성과+성찰 | 3년 누적 탐구의 결론, 한계 인식, 후속 계획 |

**⚠️ 세특 작성 주의사항:**
- 활동 나열만 있고 본인 해석이 없으면 "요약"으로 평가됨
- 자료 조사만 있고 탐구 설계가 없으면 탐구로 인정 안 됨
- 의학·생명 주제에서 윤리 고려 누락 시 평가 리스크 증가

### 🏆 면접 대비 전략

#### 입학사정관이 면접에서 체크하는 것

```
면접 평가 포인트
├── 서류 일관성 → 서류 내용과 답변이 동일한가?

**면접 답변 최적 구조:**
`문제 정의 → 기획 → 실행 → 디버깅 → 성과 → 한계 → 다음 계획`

## 📋 전체 전략 요약 — 입학사정관 체크리스트

### ✅ 공통 DO (해야 할 것)

- 학년별 성장 흐름 (탐색→심화→최적화) 구조화
- 세특·탐구·프로젝트를 **진로 키워드 중심**으로 연결
- 결과물보다 **과정 기록** (문제 정의·가설·실패·개선)
- 서류와 면접 답변의 **메시지 일치**
- 활동 전후 **비교 지표 수치**로 제시

### ❌ 공통 DON'T (하지 말 것)

- 화려한 스펙의 단순 나열 (산만한 인상)
- AI 도구 활용 사실 숨김 (검증 과정 명시가 안전)
- 대회·자격증 나열 (전공 역량 전환 경험 없음)
- 면접 과장 표현 (추가 질문에서 무너짐)
- 지원 마감·제출 형식 누락 (평가 이전 단계 탈락)
